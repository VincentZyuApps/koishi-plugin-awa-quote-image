# QQ 消息气泡风格渲染功能实施计划

**创建日期**: 2026-04-09  
**目标**: 新增模仿 QQ 默认消息气泡的图片渲染样式，支持群头衔显示

---

## 📋 需求概述

### 核心功能
1. 新增两种图片样式：`QQ_BUBBLE_LIGHT`（白天模式）和 `QQ_BUBBLE_DARK`（黑夜模式）
2. 模仿 QQ 默认消息气泡的视觉效果
3. **仅在 OneBot 平台且使用此新样式时**，支持显示群头衔标签

### OneBot API 数据结构参考

调用 `session.onebot.getGroupMemberInfo(group_id, user_id)` 返回的关键字段：

```json
{
  "group_id": 1085190201,
  "user_id": 1830540513,
  "nickname": "VincentZyu",           // QQ账号名
  "card": "https://vincentzyu233.github.io/VincentZyu233/",  // 群昵称（群名片）
  "sex": "unknown",
  "age": 21,
  "area": "",
  "level": "20",                       // 群等级
  "qq_level": 0,
  "join_time": 1772535561,
  "last_sent_time": 1775725116,
  "title_expire_time": 0,
  "unfriendly": false,
  "card_changeable": true,
  "is_robot": false,
  "shut_up_timestamp": 0,
  "role": "admin",                     // 角色："owner" | "admin" | "member"
  "title": "VincentZyuVincentZ"        // 群头衔（自定义头衔），空字符串表示无头衔
}
```

**关键字段说明**：
- `role`: 用户在群中的角色
  - `"owner"` - 群主
  - `"admin"` - 管理员  
  - `"member"` - 普通成员
- `title`: 自定义群头衔（可能为空字符串 `""`）
- `card`: 群昵称/群名片（已在现有 `onebotNameStyle` 配置中使用）
- `nickname`: QQ账号原始名称

### 群头衔显示规则

| 用户角色 (`role`) | 有自定义头衔 (`title` 非空) | 无自定义头衔 (`title` 为空) | 标签颜色 | 背景色 |
|------------------|---------------------------|---------------------------|---------|--------|
| 群主 (`"owner"`) | 显示 `title` 内容 | 显示硬编码 **"群主"** | 🟠 橙色 (#FF8C00) | rgba(255, 140, 0, 0.15) |
| 管理员 (`"admin"`) | 显示 `title` 内容 | 显示硬编码 **"管理员"** | 🔵 蓝色 (#1E90FF) | rgba(30, 144, 255, 0.15) |
| 普通成员 (`"member"`) | 显示 `title` 内容 | **不显示任何标签** | 🟣 紫色 (#9370DB) | rgba(147, 112, 219, 0.15) |

**重要说明**：
- 普通成员如果没有自定义头衔（`title === ""`），则完全不渲染头衔标签区域
- 群主和管理员即使没有自定义头衔，也会显示默认的"群主"或"管理员"标识

### 用户名显示逻辑
- 遵循现有的 `onebotNameStyle` 配置项（使用 `card` 和 `nickname` 字段）
- 与非 OneBot 平台保持一致
- **不受本新功能影响**，保持原有行为

### Step 3: 业务逻辑层改造 ([src/index.ts](../src/index.ts) - `do_aqt` 函数)

#### 任务清单
- [ ] 检测当前是否使用 QQ 气泡样式
- [ ] 如果是 OneBot 平台且启用了群头衔显示，调用 API 获取群成员信息
- [ ] 根据角色和头衔构建 `groupTitleInfo` 对象
- [ ] 将 `groupTitleInfo` 传递给 `renderQuoteImage`

#### 代码插入位置
在获取 `usernameArg` 之后，获取 `avatar_base64` 之前插入以下逻辑：

```typescript
// ========== 新增：获取群头衔信息（仅 QQ 气泡样式 + OneBot 平台）==========
let groupTitleInfo = null;

const isQqBubbleStyle = selectedStyleDetailObj.styleKey === 'QQ_BUBBLE_LIGHT' 
                     || selectedStyleDetailObj.styleKey === 'QQ_BUBBLE_DARK';

if (session.onebot && isQqBubbleStyle && config.showGroupTitleInQqBubble) {
    try {
        // 调用 OneBot API 获取群成员详细信息
        const memberInfo = await session.onebot.getGroupMemberInfo(
            session.quote.guild.id,   // 群号
            session.quote.user.id     // 用户QQ号
        );
        
        // 提取关键字段
        const role = memberInfo.role;           // "owner" | "admin" | "member"
        const customTitle = memberInfo.title;   // 自定义头衔（可能为空字符串 ""）
        
        // 根据角色和头衔构建显示信息
        if (role === 'owner') {
            // 群主：有头衔显示头衔，无头衔显示"群主"
            groupTitleInfo = {
                text: customTitle || '群主',
                color: '#FF8C00',               // 橙色
                bgColor: 'rgba(255, 140, 0, 0.15)'
            };
        } else if (role === 'admin') {
            // 管理员：有头衔显示头衔，无头衔显示"管理员"
            groupTitleInfo = {
                text: customTitle || '管理员',
                color: '#1E90FF',               // 蓝色
                bgColor: 'rgba(30, 144, 255, 0.15)'
            };
        } else if (customTitle && customTitle.length > 0) {
            // 普通成员：只有有自定义头衔时才显示
            groupTitleInfo = {
                text: customTitle,
                color: '#9370DB',               // 紫色
                bgColor: 'rgba(147, 112, 219, 0.15)'
            };
        }
        // 普通成员且无头衔：groupTitleInfo 保持为 null，不渲染任何标签
        
        // 调试日志
        if (config.verboseConsoleLog || options.verbose) {
            ctx.logger.info(`[${PLUGIN_NAME}] 群头衔信息: ${JSON.stringify({
                role, 
                customTitle, 
                groupTitleInfo
            })}`);
        }
    } catch (error) {
        ctx.logger.warn(`[${PLUGIN_NAME}] 获取群成员信息失败: ${error.message}`);
        // 失败时不中断流程，groupTitleInfo 保持为 null
    }
}
// ====================================================================
```

**关键说明**：
- `memberInfo.title` 可能是空字符串 `""`，需要用 `||` 或长度检查来判断是否有头衔
- 群主和管理员的默认文本是硬编码的中文字符串
- 普通成员无头衔时完全不渲染标签区域（`groupTitleInfo` 保持 `null`）

## ⚠️ 注意事项

### API 字段映射确认
根据 OneBot 标准 API，`getGroupMemberInfo` 返回的关键字段：
- `role`: `"owner"` | `"admin"` | `"member"` - 用户角色
- `title`: `string` - 自定义群头衔（**可能为空字符串 `""`**）
- `card`: `string` - 群昵称/群名片（已在现有逻辑中使用）
- `nickname`: `string` - QQ账号原始名称

**重要**：判断是否有头衔时应使用 `title && title.length > 0`，而非简单的 truthy 检查。

### 性能考虑
- **API调用开销**：每次渲染需额外调用一次 `getGroupMemberInfo`，预计增加 50-200ms 延迟
- **优化建议**：
  - 在日志中记录 API 调用耗时
  - 考虑未来引入缓存机制（相同用户在短时间内重复查询）
  - 设置合理的超时时间（建议 5-10 秒）

### 兼容性
- **非 OneBot 平台**：`session.onebot` 为 `undefined`，代码已通过条件检查保护
- **旧版 Koishi**：确保 `session.quote.guild.id` 和 `session.quote.user.id` 可用
- **API 版本差异**：不同 OneBot 实现可能返回略有不同的字段，需做好容错

### 错误处理
- **API 调用失败场景**：
  - 网络超时
  - Bot 无权限获取群成员信息
  - 用户已退群
  - 群不存在
  
- **处理策略**：
  - 捕获异常并记录警告日志
  - **不中断渲染流程**，`groupTitleInfo` 保持 `null`
  - 最终图片不包含头衔标签（降级方案）

### CSS 布局细节
- **头衔标签定位**：使用 Flexbox 垂直排列，确保在不同屏幕尺寸下稳定
- **气泡箭头对齐**：通过绝对定位 + CSS三角形实现，需精确计算 `top` 值以匹配头像中心
- **响应式适配**：虽然图片宽度固定，但内容区域应能自适应不同长度的文本

### 已知限制
1. **仅支持 OneBot 平台**：其他适配器无法获取群头衔信息
2. **静态样式**：头衔颜色和硬编码文本暂不支持自定义
3. **无缓存机制**：每次请求都重新调用 API（后续可优化）

### 未来优化方向
1. ✨ **自定义主题**：允许用户配置头衔颜色、气泡样式等参数
2. ✨ **缓存机制**：对同一用户的群信息进行短期缓存（如 5 分钟）
3. ✨ **更多角色支持**：如"荣誉管理员"等特殊身份
4. ✨ **动态背景图**：支持用户上传自定义气泡背景图片
5. ✨ **动画效果**：Puppeteer 支持生成 GIF/WebP 动图（高级功能）
